SpeechTTA: Make Speech in Audio Clear and Comprehensible


Abstract

Speech is an essential component of audio, which can convey information and transmit knowledge. But speech conveys more information than description text, the speech in audio generated by traditional text-to-audio models relying on description text is unclear and incomprehensible. To address this issue, we propose a novel two-stage text-to-audio model, SpeechTTA, equipped with speech generation ability from a text-to-speech model and content generation ability from a large language model. For stage one, we make the speech in audio clear by incorporating content text in speech of audio as supplements to description text. For stage two, we make the speech in audio comprehensible by generating content text based on description text, which is semantically closer to audio due to relying on the large language model. Since the lack of suitable datasets and evaluation metrics, we construct a dataset comprising description text and corresponding content text and design evaluation metrics to assess speech in audio from multiple perspectives. Experimental results demonstrate that through the two-stage process, SpeechTTA can generate clear and comprehensible speech in audio, which is semantically matched with the audio. The demo page of SpeechTTA is available.

1. Dataset

In order to design a dataset for stage two, we determine whether the intention of the description text needs to generate audio including speech based on the sound event labels in the AudioCondition that is a dataset designed for multi condition controlled audio generation. For the description text that requires speech in audio, we design a special prompt with description text to instruct the LLM (GPT-3.5-TURBO) to generate content text and pair the description text with the content text from the response to construct a new dataset, AudioContent. According to the AudioCondition, we split AudioContent into three sets: 89557 pairs for training, 1398 pairs for validation, and 1110 pairs for testing. The generation pipeline and AudioContent are publicly available.

Note that we release all of the test set. For the description text, you can find in the "captions" of the dataset. For the content text, you can find in the "answer" of the dataset.

3. Audio Samples

The following parts are the audio samples from SpeechTTA, AudioLDM, Tango, and recording (GT). We demonstrate from the following three aspects:

Require Speech: we aim to show that we can generate clear speech in audio while ensuring that speech matches the corresponding audio with the description text requiring speech.

Require Different Speakers: we aim to show that we can generate clear speech in audio spoken by different speakers with different description text.

Require No Speech: we aim to show that we can generate no speech in audio with the description text requiring no speech.
